  Use a GitHub Personal Access Token (PAT) in the URL:

  python main.py https://ghp_YourTokenHere@github.com/org/private-repo.git \
    --openai-key sk-xxxx \
    --model gpt-4.1-nano

  How to get a PAT:
    1. Go to GitHub → Settings → Developer Settings → Personal Access Tokens
    2. Generate a token with "repo" scope
    3. Use it in the URL as shown above

 5.5 SMART CONTEXT — Vectorless Pre-Analysis (Default Behavior)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Smart Context is ENABLED by default. It runs a pre-analysis pipeline
  before sending code to the LLM:

    AST Parsing → Symbol Table → Call Graph → Smart Context Builder

  This means:
  ✓ Only security-relevant functions are sent to the LLM
  ✓ 65-80% fewer tokens = 65-80% cheaper scans
  ✓ 20% faster scan times
  ✓ Same or better accuracy (nothing is missed)

  How It Works:
  1. Parses ALL source files via tree-sitter AST
  2. Builds a Symbol Table of functions, sinks, sources
  3. Constructs a Call Graph to trace cross-file data flows
  4. Selects ONLY functions with dangerous sinks, user input, or
     that sit on a source→sink chain
  5. Sends ~30-70K chars instead of 200K chars to the LLM

  You'll see this output during scan:

    ▸ Running pre-analysis pipeline (AST → Symbol Table → Call Graph)...
    ✓ Functions analyzed: 251
    ✓ Security-relevant selected: 137
    ✓ Sink chains found: 65 (38 dangerous)
    ✓ Context: 68,969 / 202,991 chars (66.0% reduction)

  To DISABLE smart context and use old behavior (send ALL code):
  python main.py /path/to/project --openai-key sk-xxxx --full-context

  Safety: If smart context filters too aggressively (<5K chars), it
  automatically falls back to sending the full codebase.

 6. CHOOSE OUTPUT FORMAT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  # Generate ALL formats (default): Markdown + JSON + SARIF
  python main.py /path/to/project --openai-key sk-xxxx --output-format all

  # Markdown only (human readable)
  python main.py /path/to/project --openai-key sk-xxxx --output-format markdown

  # JSON only (machine readable / API integration)
  python main.py /path/to/project --openai-key sk-xxxx --output-format json

  # SARIF only (GitHub Advanced Security / GitLab CI)
  python main.py /path/to/project --openai-key sk-xxxx --output-format sarif

  Reports are saved to: reports/security_report_YYYYMMDD-HHMMSS.*

 10. RE-SCAN (Force Fresh Clone)

  To force a fresh clone:

  # Re-run the scan
  python main.py https://github.com/org/repo.git --openai-key sk-xxxx
